## Message envelope

Wrap mesh messages in an `<agent-msg>` envelope so the recipient can identify, route, and reply to them:

```xml
<agent-msg from="builder" to="<peer>" id="<uuid>" ts="<iso8601>">
  <context>optional short context</context>
  <request>the actual ask</request>
  <expect>the requested reply shape</expect>
</agent-msg>
```

Send it with:

```bash
herdr agent send <name> "<agent-msg>...</agent-msg>"
```

Rules:
- `from` and `to` must name the sender and recipient from the table.
- Use a fresh UUID for `id` and ISO-8601 UTC for `ts`.
- Replies reverse `from` and `to` and add `reply-to="<original-id>"`.
- Keep messages short; put durable or long material in files and send a pointer.
