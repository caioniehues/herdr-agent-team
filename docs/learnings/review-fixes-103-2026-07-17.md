# Learnings — codebase review, 11-finding fix wave, #103 takeover (2026-07-17)

- Adversarial-verify review economics: Fable finders + sonnet refuters
  produced 10/11 CONFIRMED, 0 refuted — and the single non-confirm got
  WORSE on inspection (the "dormant" stale inbox schema was a live bug in
  `jump`). Refuting a concrete file:line claim is a bounded lookup; a
  cheaper model does it reliably.
- The highest-severity bug was schema drift nobody could see: an
  all-`Option` serde struct against a renamed schema deserializes
  SUCCESSFULLY with every field `None`. Silent-garbage class — the #92
  honesty doctrine needs exact-schema fixtures on the READ side too, not
  just the write side's exact-key-set test.
- Same-file lock patterns must be reused: `fs4` advisory locking already
  lived in `idmap.rs`/`run.rs` while `inbox_write.rs` hand-rolled a
  `create_new` sentinel that orphans on SIGKILL. Ladder rung 2 ("already
  in this codebase?") applies to concurrency primitives hardest.
- Duplicated enumeration diverges fast: `pump::discover_team_dirs` vs
  `gather::list_team_dirs` split within days — #102's liveness filter
  landed on one copy only. Canonical function + delegating wrapper costs
  ~5 lines.
- `CommandExt::exec` takeover (#103) is the right shape for "this pane
  becomes the lead": same pid, shell regains control on claude exit,
  `--resume` passes through. `--split` stays first-arg-only so claude's
  own flags are never swallowed.
- zsh wrapper gating on `HERDR_PANE_ID` cleanly splits worlds: inside
  herdr → shim lead, outside → plain claude. One-off zsh autocorrect
  prompt on first use (`.claude` dir in cwd); does not recur.
- HANDOFF.md staleness is maximally harmful when CLAUDE.md's read order
  lists it FIRST — rewrite-at-release needs to be part of the release
  ritual, not a review finding.
