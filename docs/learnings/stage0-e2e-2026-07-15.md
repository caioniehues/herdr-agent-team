# Learnings — Stage 0 E2E evidence run (2026-07-15, #46)

- **Twice-run rule paid for itself**: identical capability verdicts, but the
  run1/run2 divergence exposed the spawn-return/worker-progress race (#60)
  that a single run would have hidden.
- **The E2E found bugs the audit couldn't**: static review left the outbox
  boundary "already strong"; live drain produced `delivered` then false
  `delivery_failed` twice (#59). External-effect seams need live evidence,
  not code reading.
- **BLOCKED sentinel ≠ failure**: worker conservatism; the proof standard's
  "record the exact blocker" clause makes unexercisable sub-flows a valid
  Stage 0 outcome. Judge sentinels against acceptance criteria, not tone.
- **First-run agent onboarding is a launcher concern**: the aborted
  exploratory run stalled on Claude's bypass-permissions first-run
  confirmation; the shipped normal launcher was correct. Keep launcher
  overrides out of evidence runs.
- **Marketplace `worktree.created` hooks fired as predicted** (CLAUDE.md
  feature-interaction note): transient layout workspaces appeared; script
  had to clean only what it created. Any future E2E must scope cleanup by
  name-match, never "close everything new".
- **Report-file sentinel + background watcher worked as the monitor**: no
  polling of `done` (attention state), no lost completion.
